Project Goal

사용자 활동과 운영 정책이 함께 돌아가는 서비스 설계

01

소셜 피드 경험

팔로우 관계 기반 게시글 목록, 멤버별 프로필 피드, 좋아요, 댓글, 대댓글을 하나의 활동 흐름으로 연결했습니다.

02

실시간 이벤트

STOMP 요청을 Kafka에서 방별 순서로 버퍼링하고 MySQL 배치 저장 후 WebSocket으로 전달해 저장 안정성과 실시간성을 함께 확보했습니다.

03

운영 제어

금칙어 위험도, 자동 제재, 관리자 알림, 온라인 사용자 집계, 신규 활동 지표로 운영자가 개입할 수 있는 구조를 만들었습니다.

Core Features

커뮤니티 기능부터 부가 서비스까지 하나의 계정으로 연결

회원·인증

회원가입, 로그인, OAuth2 로그인, 메일 인증, 비밀번호·전화번호 변경, 탈퇴, 권한 관리

피드·관계

게시글 작성·수정·삭제, 이미지 업로드, 팔로우/언팔로우, 좋아요, 댓글·대댓글, 멤버 프로필 피드

채팅·알림

채팅방 생성, Kafka 방별 순서 보장, 배치 저장 후 WebSocket 전송, 채팅 이미지, SSE 알림, 읽음 처리

계좌·컬렉션·관리

계좌 생성·입금·이체, 개인 컬렉션 이미지 저장, 금칙어 신청·승인, 관리자 대시보드

Implementation Highlights

요청 처리와 부하가 큰 작업을 분리한 구현 포인트

Facade 계층

Board, Member, Chat, Notification, Account, Collection 등 복합 유스케이스를 Facade로 모아 Controller를 얇게 유지했습니다.

Kafka → MySQL 배치 영속화

roomId를 Kafka key로 사용해 방별 순서를 유지하고, 최대 100건을 JPA batch로 저장한 뒤 성공한 메시지만 WebSocket으로 전달합니다.

운영 자동화

알림 heartbeat, 오래된 알림 정리, 금칙어 누적 점수 기반 차단, 관리자 알림을 서버 내부 프로세스로 자동화했습니다.

Domain Rules

기능을 나열하는 데서 멈추지 않고 정책과 예외 처리를 코드에 반영

Account

거래 취소 정책

본인 거래 여부, 완료 상태, 취소 거래 재취소, 1일 경과 여부를 검증하고 반대 거래를 생성해 잔액을 복구합니다.

Report

신고 처리 흐름

동일 사용자의 중복 신고를 차단하고, 관리자 화면에서 신고 상태와 처리 액션, 메모를 관리할 수 있게 구성했습니다.

Moderation

금칙어 누적 제재

승인된 금칙어의 위험도 점수를 Redis에 하루 단위로 누적하고 기준을 넘으면 권한 변경과 세션 만료를 처리합니다.

Notification

실시간 알림

Kafka로 알림 이벤트를 분리하고 SSE 스트림, 읽지 않은 알림 수, 그룹별 조회를 제공해 사용자 피드백을 빠르게 전달합니다.

Service Surface

화면, REST API, WebSocket, SSE가 함께 동작하는 서비스 표면

사용자 화면

회원, 게시글, 채팅, 알림, 좋아요, 컬렉션, 계좌, 포트폴리오 화면을 Thymeleaf와 정적 리소스로 제공합니다.

/member/login · /board/main · /chat/chatList · /account
