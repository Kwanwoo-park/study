Project Goal

단순 CRUD를 넘어 운영 가능한 서비스 구조로 확장

01

실시간 경험

WebSocket 기반 채팅과 SSE 알림으로 사용자 반응성을 높였습니다.

02

비동기 처리

Kafka Producer/Consumer 구조로 알림 이벤트와 메시지 처리를 분리했습니다.

03

운영 확장성

Redis 세션 저장소, Docker, AWS EC2/S3/Load Balancer를 활용했습니다.

Implementation Highlights
